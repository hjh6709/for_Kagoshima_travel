## 2. 요구사항별 상세

### R1. 여행 핵심 정보 확인

사용자가 앱을 열었을 때 오늘 해야 할 일을 바로 알 수 있어야 한다.

수용 기준:
- 홈 화면 최상단에 오늘 날짜와 여행 일차가 표시된다.
- 다음 일정이 있으면 시간, 제목, 장소, 이동 버튼이 표시된다.
- 오늘 일정이 끝났으면 다음 날 첫 일정을 안내한다.
- 긴급 버튼은 홈 화면에서 항상 보인다.

### R2. 일정 관리

날짜별 여행 일정을 시간순으로 확인할 수 있어야 한다.

수용 기준:
- 여행 날짜를 선택할 수 있다.
- 선택한 날짜의 일정이 시간순으로 표시된다.
- 선택한 날짜 안에서 일정 순서를 직접 바꿀 수 있다.
- 진행한 일정은 완료로 표시할 수 있고 상태가 유지된다.
- 일정은 이동, 식사, 관광, 숙소, 쇼핑, 기타 유형으로 구분된다.
- 각 일정에서 지도 열기와 상세 메모 확인이 가능하다.

### R3. 지도 연결

장소별 지도 앱을 쉽게 열 수 있어야 한다.

수용 기준:
- 각 장소에는 Google Maps 링크가 있다.
- 앱 안의 지도에서 숙소, 관광지, 맛집, 이동 포인트를 볼 수 있다.
- 지도 버튼은 여행 중 누르기 쉬운 크기로 표시된다.
- 주소 텍스트를 복사할 수 있다.
- 지도 링크가 없으면 주소 검색 링크를 생성한다.
- 장소를 누르면 상세 정보와 Google Maps로 이동하는 버튼이 표시된다.

### R3-1. 장소 추천과 루트

맛집, 카페, 관광지 후보를 저장하고 여행 중 보기 쉬운 추천 루트로 묶을 수 있어야 한다.

수용 기준:
- 장소는 맛집, 카페, 관광, 쇼핑, 숙소, 교통으로 구분된다.
- 추천 장소에는 추천 이유, 영업 메모, 예산 메모, 주의사항을 기록할 수 있다.
- 루트는 장소 순서와 이동 메모를 가진다.
- 루트 내 각 장소에서 Google Maps를 열 수 있다.

### R4. 여행 정보 관리

예약과 숙소 정보를 한곳에서 확인할 수 있어야 한다.

수용 기준:
- 항공편 정보가 출국과 귀국으로 구분된다.
- 숙소 이름, 주소, 전화번호, 체크인/체크아웃 시간이 표시된다.
- 예약번호는 복사할 수 있다.
- 필요한 일본어 문장을 확인할 수 있다.

### R5. 체크리스트

여행 준비와 당일 확인 사항을 체크할 수 있어야 한다.

수용 기준:
- 출국 전, 공항, 매일 아침, 귀국 전 카테고리로 나뉜다.
- 사용자가 체크리스트 항목을 직접 추가하고 삭제할 수 있다.
- 체크 상태는 앱을 닫아도 유지된다.
- 전체 완료 수와 남은 항목 수를 표시한다.

### R6. 긴급 대응

문제가 생겼을 때 필요한 연락처와 행동 순서를 바로 확인할 수 있어야 한다.

수용 기준:
- 가족 연락처, 숙소 전화번호, 보험 연락처, 영사관 정보를 표시한다.
- 전화번호를 누르면 전화 앱이 열린다.
- 여권 분실, 길 잃음, 아픔 상황별 행동 순서를 제공한다.

### R7. 계정, 관리자와 공유

사용자는 여행 데이터를 직접 수정하고, 가족이나 동행자는 최신 정보를 확인할 수 있어야 한다.

수용 기준:
- 여행 준비자는 회원가입과 로그인 후 여행을 생성할 수 있다.
- 로그인한 여행 준비자는 일정, 장소, 루트, 긴급 정보를 생성, 수정, 삭제할 수 있다.
- 여행자 앱은 체크리스트, 일정 완료, 일정 순서, 여행 날짜를 직접 조정할 수 있다.
- 가족과 동행자는 공유 링크를 통해 최신 데이터를 확인한다.
- 공유 링크에서는 예약번호, 보험 정보 등 민감정보 표시 여부를 제어할 수 있다.

## 3. 데이터 명세

### Trip

```ts
type Trip = {
  title: string;
  startDate: string;
  endDate: string;
  travelers: string[];
  memo?: string;
};
```

### ScheduleItem

```ts
type ScheduleItem = {
  id: string;
  date: string;
  time: string;
  type: "move" | "meal" | "sightseeing" | "hotel" | "shopping" | "etc";
  title: string;
  placeName?: string;
  address?: string;
  mapUrl?: string;
  transportMemo?: string;
  reservationMemo?: string;
  parentMemo?: string;
};
```

### Place

```ts
type Place = {
  id: string;
  name: string;
  category: "hotel" | "meal" | "cafe" | "sightseeing" | "shopping" | "transport" | "etc";
  address?: string;
  latitude?: number;
  longitude?: number;
  googleMapsUrl?: string;
  recommendedReason?: string;
  openingMemo?: string;
  budgetMemo?: string;
  cautionMemo?: string;
};
```

### Route

```ts
type Route = {
  id: string;
  title: string;
  description?: string;
  placeIds: string[];
  transportMemo?: string;
  estimatedDuration?: string;
};
```

### ChecklistItem

```ts
type ChecklistItem = {
  id: string;
  category: "before" | "airport" | "daily" | "return";
  title: string;
  memo?: string;
};
```
