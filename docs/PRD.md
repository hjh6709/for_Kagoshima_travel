# 프로덕트 요구사항 문서 (PRD)

## 1. 개요

### 한 줄 정의

여행 준비자가 일정, 숙소, 골프장, 맛집, 긴급 정보를 입력하면 부모님과 가족이 공유 링크로 쉽게 확인하는 모바일 여행 도우미 앱.

### 제품 목표

- 부모님이 여행 중 “지금 어디로 가야 하지?”를 앱 첫 화면에서 바로 확인할 수 있다.
- 항공권, 숙소, 예약번호, 주소, 연락처처럼 중요한 정보를 한곳에서 확인할 수 있다.
- 일정과 긴급 정보는 인터넷이 약해도 볼 수 있다.
- 제작자는 여행 전 iPhone에서 충분히 테스트하고, 부모님 갤럭시에서 홈 화면 앱처럼 사용할 수 있게 한다.
- 제작자는 관리자 화면에서 일정, 장소, 맛집, 추천 루트를 수정할 수 있다.
- 가족은 공유 링크를 통해 최신 여행 일정과 주요 정보를 함께 확인할 수 있다.
- 포트폴리오에서는 실제 가족 문제를 해결한 PWA 사례로 보여주고, 이후 사용자 입력형 상용 서비스로 확장할 수 있다.

### 배경

6월 말 부모님이 가고시마 여행을 가신다. 부모님은 갤럭시를 사용하고, 제작자는 iPhone을 사용한다. 부모님이 현지에서 지도, 일정, 예약 정보, 긴급 연락처를 여러 앱과 메신저에서 찾는 부담을 줄이기 위해 개인용 여행 앱을 만든다.

## 2. 문제 정의

### 현재 문제

- 여행 일정, 예약 정보, 지도 링크가 여러 곳에 흩어질 수 있다.
- 부모님이 작은 글씨나 복잡한 앱 구조를 불편해할 수 있다.
- 현지에서 인터넷 연결이 불안정하면 중요한 정보를 찾기 어렵다.
- 일본어 주소, 숙소 정보, 예약번호를 필요할 때 빠르게 보여주기 어렵다.
- 여행 전후로 일정이나 맛집 후보가 바뀌면 부모님 폰에 최신 정보를 다시 전달하기 번거롭다.
- 가족이 같은 정보를 함께 봐야 할 때 메시지로 계속 공유하면 최신본 관리가 어렵다.

### 해결 방향

- 일정과 정보를 날짜순, 행동순으로 정리한다.
- 홈 화면에서 오늘의 핵심 행동을 가장 먼저 보여준다.
- 지도, 전화, 복사 같은 행동은 큰 버튼으로 제공한다.
- 긴급 정보는 하단 탭에서 항상 접근 가능하게 둔다.
- 관리자 화면에서 여행 데이터와 장소 정보를 수정하면 부모님용 앱에 최신 내용이 반영되게 한다.
- 장소는 앱 안의 지도에서 한눈에 보고, 필요하면 Google Maps로 이동해 상세 길찾기를 한다.

## 3. 사용자와 역할

| 역할 | 설명 | 주요 니즈 |
| --- | --- | --- |
| 부모님 | 여행 중 실제 앱 사용자 | 오늘 일정 확인, 지도 열기, 긴급 연락 |
| 여행 준비자 | 정보를 입력하고 공유하는 사용자 | 여행 생성, 일정/숙소/골프장/긴급 정보 입력, 공유 링크 발급 |
| 제작자 | 앱 제작 및 초기 관리자 | iPhone 테스트, 서비스 개선, 포트폴리오 정리 |
| 가족 | 공유 사용자 | 최신 일정 확인, 장소와 긴급 정보 파악 |

## 4. 성공 기준

- 부모님 갤럭시에서 홈 화면에 앱을 추가하고 실행할 수 있다.
- 홈 화면에서 다음 일정과 지도 버튼을 5초 안에 찾을 수 있다.
- 날짜별 전체 일정을 확인할 수 있다.
- 긴급 연락처와 숙소 주소를 인터넷 없이 확인할 수 있다.
- 체크리스트 완료 상태가 기기 안에 저장된다.
- 관리자가 일정을 수정하면 부모님 앱과 가족 공유 화면에 최신 내용이 반영된다.
- 지도 화면에서 일정 장소, 맛집, 숙소를 구분해서 볼 수 있다.

## 5. 범위

### Phase 1. 개인용 PWA 포함

- 홈
- 날짜별 일정
- 장소별 지도 열기
- 여행 정보
- 체크리스트
- 긴급 정보
- 오프라인 캐시
- 홈 화면 추가용 PWA 설정
- 지도 화면
- 맛집 및 추천 장소 관리
- 추천 루트 관리

### Phase 2. 사용자 입력형 Beta 포함

- 회원가입과 로그인
- 여행 생성
- 일정, 장소, 숙소, 골프장, 긴급 정보 입력/수정
- 공유 링크 생성
- 가족/부모님용 읽기 전용 공유 페이지
- 민감정보 노출 범위 제어

### 이후로 미룰 수 있는 범위

- 실시간 업데이트
- 실시간 위치 공유
- 푸시 알림
- 다국어 전체 번역
- 앱스토어와 플레이스토어 배포
- 자체 지도 엔진 제작
- 소셜 로그인
- 결제
- 복잡한 팀 권한 관리

## 6. 주요 정책

- 1차 개인용 PWA 데이터는 정적 프론트엔드 데이터 파일에서 관리한다.
- 사용자 입력형 Beta부터는 Go 백엔드와 PostgreSQL에 사용자별 여행 데이터를 저장한다.
- 부모님 앱은 조회 중심으로 만든다.
- 여행 준비자와 가족 관리자는 로그인 후 데이터를 입력하고 수정한다.
- 부모님과 동행자는 로그인 없이 공유 링크로 읽기 전용 화면을 본다.
- 체크리스트 완료 여부만 로컬 저장한다.
- 지도는 갤럭시 기준 Google Maps 링크를 우선 제공한다.
- iPhone 테스트를 위해 Apple Maps 또는 브라우저 지도 링크도 열 수 있게 고려한다.
- 전화번호, 주소, 예약번호는 복사 버튼을 제공한다.
- 장소 데이터는 이름, 주소, 좌표, 카테고리, 추천 메모, 지도 링크를 가진다.
- 추천 루트는 여러 장소를 순서대로 묶고 예상 이동 메모를 포함한다.
- 가족 공유 화면은 민감정보 노출 범위를 조절할 수 있어야 한다.
- 실제 배포 URL은 사실상 공개 URL이므로 민감정보 입력을 최소화한다.

## 7. 비기능 요구사항

- 모바일 화면 기준으로 먼저 설계한다.
- 주요 버튼 높이는 최소 44px 이상으로 한다.
- 본문 글씨는 작게 만들지 않는다.
- 핵심 정보는 높은 대비로 표시한다.
- 앱 초기 로딩 후 핵심 데이터는 오프라인에서도 표시한다.
- 배포 URL은 부모님께 메시지로 공유할 수 있어야 한다.
- 관리자 변경 사항은 새로고침 없이 가능한 한 빠르게 반영한다.
- 백엔드 장애가 있어도 마지막으로 불러온 핵심 여행 정보는 볼 수 있어야 한다.
