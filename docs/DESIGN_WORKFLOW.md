# 디자인 워크플로우

이 문서는 Claude Code, Gemini Pro, 참고 이미지, 디자인 체크리스트를 활용해 가고시마 여행 앱의 화면 품질을 높이는 방법을 정리한다.

## 1. 디자인 목표

이 프로젝트는 고급 레스토랑 랜딩페이지처럼 화려한 웹사이트가 아니라, 부모님이 여행 중 실제로 쓰는 모바일 도구다.

따라서 디자인 기준은 아래 순서로 둔다.

1. 잘 보이는가
2. 헷갈리지 않는가
3. 손가락으로 누르기 쉬운가
4. 인터넷이 불안해도 핵심 정보가 남는가
5. 가족이 만든 앱처럼 따뜻하지만, 장난스럽지 않은가
6. 지나치게 앱 같지 않은 웹페이지가 아니라, 홈 화면에 추가했을 때 실제 앱처럼 느껴지는가

## 2. AI 도구 역할 분담

### Claude Code

Claude Code는 코드 수정과 구조 판단에 사용한다.

주요 용도:
- React 컴포넌트 구현
- PWA 설정
- 접근성/모바일 UX 점검
- 코드 리뷰 반영
- 디자인 변경을 실제 CSS와 컴포넌트로 구현

Claude에게 요청할 때는 “예쁘게 해줘”보다 의도를 구체적으로 준다.

좋은 요청:

```text
부모님이 여행 중 급하게 봐도 다음 행동을 바로 알 수 있게 홈 화면의 정보 위계를 개선해줘.
버튼은 더 크고 명확하게 만들되, 화면이 복잡해지지 않게 해줘.
```

피해야 할 요청:

```text
10,000달러짜리 웹사이트처럼 만들어줘.
```

### Gemini Pro

Gemini Pro는 시각 방향을 잡거나 비교 평가에 사용한다.

주요 용도:
- 참고 이미지 분석
- 컬러/타이포그래피 방향 제안
- 모바일 화면 스크린샷 평가
- “부모님용 앱으로 보이는지” 관점의 피드백
- 한국어 문구를 더 자연스럽고 짧게 다듬기

Gemini에게 줄 만한 요청:

```text
이 화면은 부모님이 일본 여행 중 사용할 PWA 앱입니다.
시각적으로 예쁜지보다, 정보가 잘 보이고 긴급 상황에서 헷갈리지 않는지 평가해주세요.
개선점을 5개 이하로 우선순위별로 제안해주세요.
```

## 3. 참고 이미지 사용 원칙

참고 이미지는 복사 대상이 아니라 취향과 방향을 설명하기 위한 재료다.

추천 참고 유형:
- 모바일 여행 일정 앱
- 항공권/호텔 예약 앱의 일정 상세 화면
- Google Maps 장소 상세 UI
- Apple Wallet 또는 boarding pass 스타일의 정보 카드
- 일본 여행 가이드 앱의 장소 목록

피할 참고 유형:
- 과한 마케팅 랜딩페이지
- 어두운 럭셔리 레스토랑 사이트
- 애니메이션이 많은 포트폴리오 사이트
- 텍스트가 작고 장식이 많은 디자인

## 4. 디자인 체크리스트

영상의 고급 웹사이트 기준을 이 앱에 맞게 바꾸면 아래와 같다.

| 항목 | 이 앱에서의 의미 |
| --- | --- |
| 관점 | 부모님 여행 중 불안과 혼란을 줄이는 도구 |
| 타이포그래피 | 큰 글씨, 짧은 문장, 선명한 숫자와 시간 |
| 색상 | 밝은 배경, 높은 대비, 긴급 정보만 제한적 빨강 |
| 위계 | 다음 일정, 지도, 긴급 연락이 가장 먼저 보임 |
| 이미지 | 실제 장소 이해에 도움이 될 때만 사용 |
| 모션 | 없어도 됨. 있더라도 상태 변화와 피드백 중심 |
| 모바일 | 데스크톱 축소판이 아니라 모바일 우선 화면 |
| 보이지 않는 품질 | 빠른 로딩, 오프라인 캐시, 홈 화면 아이콘, 깨지지 않는 글자 |

## 5. 화면별 디자인 기준

### 홈

- 다음 일정이 가장 크게 보여야 한다.
- “지도 열기”와 “긴급 연락”은 바로 누를 수 있어야 한다.
- 오늘의 전체 일정은 요약만 보여준다.

### 일정

- 날짜 선택이 쉬워야 한다.
- 시간순으로만 보여준다.
- 일정 유형은 아이콘과 짧은 라벨로 구분한다.
- 일정 카드마다 지도 버튼이 있어야 한다.

### 지도/장소

- 앱 안에서는 장소 목록과 추천 이유를 보여준다.
- 실제 길찾기는 Google Maps로 넘긴다.
- 주소는 텍스트로도 보여줘야 한다.

### 정보

- 항공편, 숙소, 일본어 문장처럼 보여줄 정보만 담는다.
- 예약번호나 민감정보는 1차 MVP에 넣지 않는 것을 기본으로 한다.

### 체크

- 체크 항목은 손가락으로 누르기 쉬워야 한다.
- 완료 수가 보여야 한다.
- 앱을 닫아도 체크 상태가 유지되어야 한다.

### 긴급

- 가장 단순해야 한다.
- 전화 버튼이 크고 명확해야 한다.
- 행동 순서는 1, 2, 3으로 짧게 적는다.

## 6. 디자인 리뷰 요청 프롬프트

Claude Code나 Gemini Pro에 아래처럼 요청한다.

```text
이 앱은 부모님이 가고시마 여행 중 사용하는 모바일 PWA입니다.
고급 랜딩페이지처럼 화려하게 만들기보다, 여행 중 빠르게 보고 누를 수 있는 실용성과 안심감을 우선합니다.

다음 기준으로 화면을 리뷰해주세요.
1. 부모님이 다음 행동을 5초 안에 알 수 있는가
2. 글씨와 버튼이 모바일에서 충분히 큰가
3. 긴급 정보 접근이 명확한가
4. 지도/주소/전화 행동이 헷갈리지 않는가
5. 화면이 장식보다 정보 전달에 집중하는가

문제는 우선순위별로 최대 7개만 제안하고, 대규모 리디자인보다 바로 고칠 수 있는 수정안을 주세요.
```

## 7. 폴리시 패스

첫 화면 구현 후 아래 순서로 다듬는다.

1. 모바일 화면 스크린샷을 찍는다.
2. Gemini Pro에 시각 위계와 가독성을 리뷰시킨다.
3. Claude Code에 코드 수정 범위를 좁혀 반영시킨다.
4. iPhone과 갤럭시 폭 기준으로 글자 잘림을 확인한다.
5. 비행기모드에서 홈, 일정, 긴급 화면이 보이는지 확인한다.

## 8. 이번 앱에서 과한 것

아래는 1차 MVP에서 하지 않는다.

- 커서 추적 효과
- 스크롤 기반 시네마틱 히어로
- 큰 배경 영상
- 과한 마이크로 인터랙션
- 복잡한 페이지 전환 애니메이션
- 고급 랜딩페이지식 섹션 연출

부모님 여행 앱의 고급스러움은 화려함이 아니라, 필요한 정보가 조용히 잘 정리되어 있는 데서 나온다.
