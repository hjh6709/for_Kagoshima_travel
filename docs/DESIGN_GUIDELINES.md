# 모바일 UI/UX 타이포그래피 및 디자인 가이드라인

이 문서는 모바일 PWA 여행 앱의 완성도를 높이고, 시간이 지나도 촌스럽지 않은 일관성 있는 화면을 그리기 위한 **7대 디자인 원칙 및 실무 규칙**을 규정합니다. 향후 새로운 UI 컴포넌트를 추가하거나 레이아웃을 다듬을 때 이 가이드를 엄격히 준수합니다.

---

## 🎨 모바일 타이포그래피 7대 핵심 원칙

### 1. 화려함보다 심플함 (Simplicity)
- 한 제목이나 카드에 너무 많은 장식 효과(그림자, 다채로운 굵기 조합, 다중 색상)를 섞지 않습니다.
- 굵기, 색상, 그림자 등의 데코레이션은 **최대 2개까지만 혼용**하며 전체적인 통일감을 유지합니다.
- 계층 구별은 폰트 자체의 색상 대비(예: Dark Green과 Muted Gray)만으로도 충분히 구현 가능합니다.

### 2. 긴 문장은 무조건 좌측 정렬 (Left Align)
- **3줄 이상 넘어가는 모든 설명글 및 캡션은 무조건 좌측 정렬(`text-align: left`)** 합니다.
- 가운데 정렬은 줄마다 시작 지점이 달라져 시선이 흔들리고 읽기 어렵습니다. 좌측 정렬은 가독성을 극적으로 높여줍니다.

### 3. 포인트 컬러의 제한적 사용 (Color Restraint)
- 강조할 부분마다 포인트 컬러를 남용하면 오히려 전체 강조가 무력화됩니다.
- 핵심 액션 단추나 에러 경고 등 정말 중요한 정보에만 브랜드 포인트 컬러(골드/그린 등)를 제한적으로 사용합니다.
- 강조는 글자 크기 조정, Bold(700), 여백 배치만으로도 훌륭하게 설계할 수 있습니다.

### 4. 시선을 유도하는 폰트 계층화 (Typography Hierarchy)
- 사용자가 화면에 진입했을 때 자연스럽게 `제목 ➡️ 부제 ➡️ 본문 ➡️ 보조 정보` 순서로 시선을 흘려보내도록 설계해야 합니다.
- 계층 관계는 글자 크기(`font-size`), 두께(`font-weight`), 색상 투명도 대비, 그리고 명확한 마진 블록으로 형성합니다.

### 5. 폰트 패밀리 최소화 (One Family Rule)
- 한 화면에서 여러 폰트 서체를 혼용하지 않습니다.
- 이 프로젝트는 **`Pretendard` 단일 폰트 패밀리**만 기본 글꼴로 규정하여 통일성 및 가독성을 유지하며 로딩 속도를 최적화합니다.

### 6. 폰트 두께 단계 통일 (Weight System)
- 폰트의 두께는 파편화를 막기 위해 아래의 **3단계 규칙**으로 제한합니다:
  - **Regular (400)**: 본문 텍스트, 설명 캡션
  - **Medium (500)**: 중간 서브 타이틀, 폼 라벨
  - **Bold (700)**: 메인 대제목, 강조 버튼 텍스트
  - *기존의 비표준 굵기(800, 900)는 사용하지 않습니다.*

### 7. 모바일 뷰 최적화 (Mobile First Check)
- PC 화면에서 적당해 보이는 폰트도 모바일에서는 작게 보입니다.
- 주요 터치 컴포넌트(버튼, 카테고리 필터, 검색 필드)는 모바일 화면에서 충분히 크게(기본 버튼 높이 40~48px 권장) 디자인하고, 반드시 스마트폰 화면으로 최종 실물 검수합니다.

## 🛠️ 실무 적용 세부 가이드 (Reference Specs)

| 구분 | 권장 스펙 기준 | 비고 |
| :--- | :--- | :--- |
| **좌우 여백 (Margin)** | 16px / 20px / 24px 중 하나로 수렴 | 모바일 프레임 내부 기본 안쪽 여백 |
| **행간 (Line-height)** | **140% (1.4)** ~ **150% (1.5)** | 가독성 보장을 위한 줄간격 의무 규정 |
| **글자 크기 (Font-size)** | 12 / 14 / 16 / 20 / 24 / 28 / 32 / 36 / 48 px | 본문은 14px, 타이틀은 18~24px 권장 |
| **컴포넌트 높이 (Height)** | 48px / 40px / 36px | 버튼, 셀렉트박스, 인풋 박스 높이를 동일선상에 매핑 |
