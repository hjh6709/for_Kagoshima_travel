# 가고시마 부모님 여행 앱 기획서

## 1. 프로젝트 개요

6월 말 가고시마 여행을 가시는 부모님이 현지에서 길 찾기, 일정 확인, 준비물 확인, 긴급 연락을 쉽게 할 수 있도록 돕는 개인용 여행 앱을 만든다.

부모님은 안드로이드 갤럭시를 사용하지만, 제작자 본인은 iPhone으로 테스트해야 하므로 안드로이드와 iOS 양쪽에서 확인 가능한 형태를 우선 고려한다.

## 2. 핵심 목표

- 부모님이 앱을 열면 오늘 해야 할 일정과 이동 정보를 바로 볼 수 있다.
- 인터넷이 불안정해도 주요 일정, 주소, 연락처, 체크리스트는 확인할 수 있다.
- 화면과 버튼은 크고 단순하게 만든다.
- 일본어를 몰라도 숙소, 식당, 관광지, 교통 정보를 보여주거나 전달할 수 있다.
- 가족이 사전에 입력한 일정과 메모를 부모님이 헷갈리지 않게 순서대로 보여준다.

## 3. 주요 사용자

- 1차 사용자: 부모님
  - 갤럭시 사용
  - 여행 중 빠르게 보고 누를 수 있는 단순한 화면 필요
  - 작은 글씨, 복잡한 메뉴, 많은 선택지는 부담이 될 수 있음
- 2차 사용자: 제작자
  - iPhone으로 사전 테스트
  - 일정, 장소, 메모, 긴급 정보를 직접 입력하고 수정

## 4. 권장 제작 방식

### 1순위: PWA 웹앱

브라우저에서 접속하고 홈 화면에 추가해서 앱처럼 쓰는 방식이다.

장점:
- 안드로이드와 iPhone에서 모두 테스트 가능
- 앱스토어나 플레이스토어 등록 없이 바로 공유 가능
- 여행 전까지 빠르게 만들기 좋음
- 오프라인 캐시, 홈 화면 아이콘, 전체 화면 실행을 지원 가능

단점:
- 네이티브 앱보다 일부 기능 제한이 있음
- 푸시 알림이나 백그라운드 기능은 기기별 차이가 있음

### 2순위: Expo React Native 앱

실제 모바일 앱 형태로 만들 수 있는 방식이다.

장점:
- 안드로이드와 iOS 앱을 동시에 개발 가능
- 모바일 UI와 기기 기능을 더 자연스럽게 사용 가능

단점:
- 설치, 빌드, 배포 과정이 PWA보다 복잡함
- 부모님 폰에 설치하려면 추가 절차가 필요할 수 있음

### 초기 결론

6월 말 여행용 개인 앱이라면 `PWA 웹앱`으로 먼저 만드는 것이 현실적이다. 이후 필요하면 Expo 앱으로 확장한다.

## 5. MVP 범위

여행 전 반드시 완성할 최소 기능이다.

### 홈

- 오늘 날짜와 여행 일차 표시
- 오늘 일정 요약
- 가장 중요한 다음 행동 표시
- 긴급 버튼 바로가기

### 일정

- 날짜별 일정 목록
- 시간, 장소명, 주소, 이동 방법, 메모 표시
- 각 일정에서 지도 열기 버튼 제공
- 식사, 관광, 이동, 숙소 체크 구분

### 지도 바로가기

- Google Maps 또는 Apple Maps로 장소 열기
- 부모님 갤럭시 기준으로 Google Maps 우선
- iPhone 테스트를 위해 Apple Maps 링크도 선택 가능하게 고려

### 여행 정보

- 항공편 정보
- 숙소 정보
- 예약번호
- 체크인/체크아웃 시간
- 필요한 일본어 문장
- 현지 교통 메모

### 체크리스트

- 출국 전 준비물
- 공항에서 할 일
- 매일 아침 확인할 것
- 귀국 전 확인할 것

### 긴급

- 가족 연락처
- 숙소 전화번호와 주소
- 한국 영사관/긴급 연락처
- 여권 분실 시 행동 순서
- 여행자보험 정보

## 6. 화면 구조

하단 탭 5개를 기본 구조로 한다.

- 홈
- 일정
- 지도
- 체크
- 긴급

부모님용 앱이므로 메뉴명은 짧게 유지하고, 각 화면의 첫 번째 정보가 바로 눈에 들어오도록 구성한다.

## 7. 데이터 구조 초안

### Trip

- 여행명
- 시작일
- 종료일
- 인원
- 메모

### Day

- 날짜
- 여행 일차
- 제목
- 날씨 메모
- 일정 목록

### ScheduleItem

- 시간
- 종류: 이동, 식사, 관광, 숙소, 쇼핑, 기타
- 제목
- 장소명
- 주소
- 지도 링크
- 이동 방법
- 예약 정보
- 부모님께 보여줄 메모

### EmergencyInfo

- 이름
- 설명
- 전화번호
- 주소
- 링크

### ChecklistItem

- 카테고리
- 제목
- 완료 여부
- 메모

## 8. UX 원칙

- 글씨는 기본적으로 크게 사용한다.
- 버튼은 손가락으로 누르기 쉽게 충분히 크게 만든다.
- 한 화면에 너무 많은 정보를 넣지 않는다.
- 중요한 정보는 색상, 아이콘, 순서로 구분한다.
- 일본어, 주소, 예약번호처럼 복사해야 할 가능성이 있는 텍스트는 복사 버튼을 둔다.
- 긴급 화면은 어디서든 1번 탭으로 접근 가능해야 한다.
- 일정 화면은 시간순으로만 보여줘서 판단 부담을 줄인다.

## 9. 오프라인 전략

- 앱 자체는 홈 화면에 추가해서 실행 가능하게 한다.
- 일정, 체크리스트, 긴급 정보는 로컬 데이터로 포함한다.
- 지도 앱 연결은 인터넷이 필요할 수 있음을 안내한다.
- 여행 전 부모님 폰에서 앱을 한 번 열어 캐시되도록 한다.

## 10. 디자인 방향

- 부모님용이므로 예쁜 여행 매거진보다 실용적인 여행 도구에 가깝게 만든다.
- 색상은 밝은 배경, 높은 대비, 구분이 쉬운 포인트 컬러를 사용한다.
- 각 일정은 카드 형태로 분리하되 둥근 정도는 과하지 않게 한다.
- 아이콘을 적극적으로 사용해서 읽기 전에 의미가 보이게 한다.
- 긴급 정보는 빨간색 계열을 제한적으로 사용해 위급함을 명확히 한다.

## 11. 기술 스택 초안

- 프론트엔드: React
- 빌드 도구: Vite
- 언어: TypeScript
- 스타일: CSS 또는 Tailwind CSS
- 앱 형태: PWA
- 1차 데이터: 정적 TypeScript 데이터
- 1차 배포: Vercel Hobby 무료 플랜
- 2차 백엔드: Go API 서버
- 2차 데이터베이스: PostgreSQL
- 2차 인증: Go API에서 세션 또는 JWT로 구현
- 2차 실시간 업데이트: 1차는 짧은 주기 갱신, 이후 SSE 또는 WebSocket 검토
- 지도 표시: Google Maps 외부 링크 우선
- 지도 링크: Google Maps 웹 링크 우선
- 배포: Vercel Hobby 무료 플랜 우선

## 11-1. 백엔드가 필요한 이유

아래 기능을 포함하려면 정적 데이터만으로는 부족하다.

- 일정 생성, 수정, 삭제
- 가족 공유 페이지
- 관리자 페이지
- 부모님 앱의 최신 데이터 반영
- 맛집, 관광지, 숙소, 교통 포인트 관리
- 추천 루트 관리
- 민감정보 공개 범위 제어

다만 현재는 비용을 최대한 줄이는 것이 중요하므로, 1차 여행 MVP는 백엔드 없이 정적 PWA로 완성한다. Go 백엔드는 2차 확장 준비물로 유지한다.

## 11-2. 지도 전략

- 1차에서는 앱 안에 장소 목록과 지도 열기 버튼을 제공한다.
- 장소를 누르면 상세 카드에서 주소, 메모, 추천 이유를 보여준다.
- 길찾기는 Google Maps로 이동해서 처리한다.
- 자체 길찾기 엔진은 만들지 않는다.
- 맛집과 추천 루트는 관리자가 입력하고, 추후 실제 현지 정보로 보강한다.

## 12. 개발 순서

1. 기획서 확정
2. 샘플 일정 데이터 작성
3. React + Vite 프로젝트 생성
4. 모바일 우선 레이아웃 구현
5. 홈, 일정, 지도 링크, 장소 추천, 체크리스트, 긴급 화면 구현
6. PWA 설정 추가
7. 실제 여행 일정과 예약 정보 입력
8. 갤럭시와 iPhone에서 접속 테스트
9. Vercel Hobby 무료 배포
10. 오프라인 동작 확인
11. 부모님 폰 홈 화면에 추가
12. 필요 시 Go 백엔드와 PostgreSQL 확장

## 13. 확인이 필요한 정보

- 정확한 여행 기간
- 항공편 시간과 공항
- 숙소 이름, 주소, 전화번호, 체크인 시간
- 날짜별 방문지
- 이동수단: 렌터카, 버스, 택시, 전철 여부
- 예약한 식당이나 투어 정보
- 여행자보험 정보
- 부모님이 자주 연락할 가족 연락처
- 부모님이 현지에서 가장 걱정하시는 상황

## 14. 다음 단계 제안

우선 샘플 데이터로 앱 골격을 만든 뒤, 실제 여행 정보가 확정되는 대로 JSON 데이터를 교체한다. 첫 구현 목표는 부모님 폰에서 홈 화면에 추가해 사용할 수 있는 PWA 프로토타입이다.
